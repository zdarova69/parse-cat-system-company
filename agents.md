Тестовое задание на аналитика данных в Lead Sniper (Python / JavaScript)
Цель задания
Проверить, что вы умеете:
находить источники данных о компаниях;
вытаскивать данные программно (API / парсинг / работа с открытыми датасетами);
приводить данные к аккуратному табличному виду;
документировать свой подход.


Задача максимально приближена к реальной работе у нас.

1. Задача
Собрать базу российских компаний, которые:
используют CAT-систему / TMS / платформу локализации,
и имеют годовую выручку от 100 млн ₽ (по последнему доступному году).


Что считаем CAT-системой в рамках задания
Под «CAT-системами» понимаем решения для компьютерной поддержки переводов и/или управления процессом локализации, где присутствуют хотя бы часть типичных функций:
Translation Memory (память переводов),
терминологические базы,
сегментация/редактирование,
QA/LQA,
совместная работа переводчиков,
управление проектами перевода (workflow, роли, интеграции),
TMS-модули.


Важно: в таблице должен быть явный признак/доказательство, что CAT-система у компании есть.

2. Требования к результату
2.1. Таблица (CSV)
Файл companies.csv с минимум 50+ компаниями, лучше 80+.
Каждая строка — одна компания.
Обязательные поля (колонки)
inn — ИНН компании
name — полное или сокращённое название
revenue — выручка за год (в рублях, числом)
site — основной сайт компании
cat_evidence — короткое подтверждение признака CAT (например: “использование продукта на сайте”, “упоминание TM/TMS”)
source — откуда взяты данные (например: rusprofile, list-org, open_data, custom_parser, company_site)


Желательные поля (плюс к вам)
cat_product — название CAT/TMS/локализационного продукта или платформы (если известно)
employees — оценка количества сотрудников (числом)
okved_main — основной ОКВЭД (если несколько — укажите главный)


Важно: данные должны быть приведены к единому формату (без мусора, лишних пробелов, диапазонов вместо чисел и т.п.).

3. Требования к коду
Выложить код в публичный репозиторий на GitHub.
Структура репозитория (пример)
README.md
requirements.txt или package.json
src/ (основной код)
data/companies.csv (итоговый файл)
при необходимости — config.example.json / .env.example

Что должно быть в коде
Скрипт(ы) для сбора данных.
Python или Node.js — на выбор.
Можно использовать:
открытые базы данных;
rusprofile, list-org и аналоги;
парсинг сайтов компаний;
отраслевые каталоги/подборки;
HH / Telegram / другие источники ;
LLM / API LLM (для нормализации описаний и классификации “есть ли признаки CAT”).


Обязательно: хотя бы один источник должен быть обработан программно, а не вручную.

4. Минимальная обработка данных
Нужно сделать:
очистку сырого вывода;
нормализацию форматов;
фильтрацию компаний до критериев:
Россия;
использование CAT-системы;
выручка ≥ 100 млн ₽ по последнему доступному году.



5. Повторяемость
Скрипты должны запускаться одной-двумя командами:
python main.py
 или
npm start / node src/index.js


В README.md кратко описать:
какие источники использовали;
как запустить код;
какие ограничения / упрощения заложили.

Что получилось / что не получилось

Роль LLM

нормализация описаний,

Объём отчёта — 0.5–1 страница текста, без воды.

7. Как мы будем оценивать
Качество данных
релевантность (это действительно компании с CAT-системами, а не просто случайные переводчики без платформ);
корректность фильтра по выручке ≥ 100 млн ₽;
чистота и единый формат полей;
наличие понятного cat_evidence.


Код и архитектура
читаемость, понятные имена переменных и файлов;
базовая структура (разделение на «сбор → очистка → фильтрация → выгрузка»);
простая, но вменяемая обработка ошибок.


Повторяемость

Аналитика и мышление

качество определения признака “есть CAT-система”.


Инициатива
